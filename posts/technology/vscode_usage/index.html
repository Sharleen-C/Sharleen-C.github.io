VSCode使用相关问题#

修改文件图标#

Extensions -> VSCode Icons -> Install -> Apply

安装Nodejs#

官网下载对应版本安装包

指定路径进行下载，并将下载路径写入环境变量

D:\NodeJs\install

1. 查看npm默认存放位置#

使用npm get prefix查看npm全局模块的存放路径

1npm get prefix
2>>> C:\Users\xxx\AppData\Roaming\npm

使用npm get cache查看npm缓存默认存放路径

1npm get cache
2>>> C:\Users\xxx\AppData\Local\npm-cache

2. 修改默认文件夹#

在 nodejs 安装目录下，创建 “node_global” 和 “node_cache” 两个文件夹

设置全局模块的安装路径到 “node_global” 文件夹，

npm config set prefix "D:\NodeJs\install\node_global"

设置缓存到 “node_cache” 文件夹

npm config set cache "D:\NodeJs\install\node_cache"

由于 node 全局模块大多数都是可以通过命令行访问的，还要把【node_global】的路径“E:\devTools\nodejs\node_global”加入到【系统变量 】下的【PATH】 变量中，方便直接使用命令行运行

3. 测试默认位置是否更改成功#

经过上面的步骤，nodejs下载的模块就会自动下载到我们自定义的目录，接下来我们测试一下是否更改成功。输入下面的命令：

npm install express -g
or
npm install express --global

若执行命令npm install express -g出现如下报错：

是由于对文件夹操作的权限不够，右击安装路径文件夹（D:\NodeJs\install）->属性->安全，点击编辑，将所有权限都✔即可

如果依旧报错，就将D:\NodeJs\install\node_modules D:\NodeJs\install\node_cache D:\NodeJs\install\node_global同样赋予所有权限

4. 设置淘宝镜像#

查看当前使用的镜像路径

1npm config get registry
2>>> https://registry.npmjs.org/

更换为国内的淘宝镜像

npm config set registry https://registry.npmmirror.com

再次检查当前的镜像

1npm config get registry
2>>> https://registry.npmmirror.com

说明：由于npm的服务器在海外，所以访问速度比较慢，访问不稳定 ，cnpm的服务器是由淘宝团队提供，服务器在国内，cnpm是npm镜像，一般会同步更新，相差在10分钟，所以cnpm在安装一些软件时候会比较有优势。但是cnpm一般只用于模块安装，在项目创建与卸载等相关操作时仍使用npm。

全局安装基于淘宝源的cnpm

npm install -g cnpm --registry=https://registry.npmmirror.com

执行命令查看cnpm是否安装成功

1cnpm -v

系统禁止运行脚本#

在 Vs Code 终端输入 vue init webpack app， 创建一个名为 app 的 Vue 项目时报错如下： 无法加载文件 D:\nodejs\node_global\webpack.ps1，因为在此系统上禁止运行脚本，有关详细信息，请参阅 https:/go.microsoft.com/fwlink/?LinkID=135170

解决报错： （1）以管理员身份运行vs code

（2）在终端执行：get-ExecutionPolicy，显示Restricted（表示状态是禁止的）

（3）在终端执行：set-ExecutionPolicy RemoteSigned

（4）在终端执行：get-ExecutionPolicy，显示RemoteSigned

此时再使用“vue init webpack 项目名称”创建项目，不会报错

1get-ExecutionPolicy
2>>> Restricted
3set-ExecutionPolicy RemoteSigned
4get-ExecutionPolicy
5>>> RemoteSigned

webpack安装#

在安装webpack前，你必须为本机安装 node.js

由于webpack4后的版本，命令迁移至webpack-cli，因此还需安装webpack-cli

全局安装#

 npm install webpack webpack-cli -g --save-dev

安装后执行

1webpack -v

正常显示说明安装好了

项目安装#

npm install webpack webpack-cli --save-dev

项目下多了node_moudle文件 package_lock.json文件；说明成功安装

1npm init

一路回车，项目出现package.json
